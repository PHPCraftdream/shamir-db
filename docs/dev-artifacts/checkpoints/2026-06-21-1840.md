## Decisions

- Полностью убрали FULL bench mode (hardcoded `is_full() = false`) вместо двойного env-var
  гейта — пользователь явно попросил "чтобы никак нельзя было его запустить без правок кода".
- Удалили nebari/persy/canopy/redb полностью (а не оставили redb как infrastructure)
  по решению пользователя ("так заменим его на fjall и удалим"). Переписали native redb API
  на fjall, добавив parking_lot::Mutex для атомарности RMW (fjall не имеет nested transactions).
- В fjall портах используем `db.batch()` для cross-keyspace atomic writes (insert в
  user_directory — username + user_id index) — это аналог redb's atomic write transaction.
- `Durability::Immediate` (redb) → `db.persist(PersistMode::SyncAll)` (fjall) после каждого
  accepted write для spec §3.5/§6.2/§B13 NORMATIVE fsync.
- На-диск ext'ы убраны: `*.redb` → directory без extension (fjall - directory-based).
  Тесты обновлены под новый shape.
