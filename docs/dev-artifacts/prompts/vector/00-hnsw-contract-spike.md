## Зачем этот лист

Дальнейшие фазы (persist P2, filtered ANN P3, quantization P5) опираются на
конкретный API `hnsw_rs`. Источники ПРОТИВОРЕЧАТ, существует ли в версии 0.3.x
то, что нам нужно (`file_dump`/`HnswIo`/`parallel_insert`/`search_filter`). Этот
лист — спайк: установить ГРОУНД-ТРУС по реальному API 0.3.4 и закрепить его
контракт-тестами, чтобы поздние фазы строились на проверенных фактах, а не на
догадках.

## Задача

1. **Пин версии + зависимость.** В `crates/shamir-index/Cargo.toml`:
   - сменить `hnsw_rs = "0.3"` на `hnsw_rs = "=0.3.4"` (стабильность
     dump-формата между патчами не гарантирована);
   - добавить `crc32fast` (уже в workspace — сверь версию по другому
     crate, напр. shamir-wal; используй ту же). Он понадобится фазе P2, здесь
     заводим зависимость заранее одной правкой Cargo.toml.
2. **Сначала — установить реальный API 0.3.4.** Прогони `cargo doc -p hnsw_rs`
   ИЛИ загляни в исходники крейта в `~/.cargo/registry/.../hnsw_rs-0.3.4/`
   (grep по `pub fn`), чтобы точно знать сигнатуры. НЕ доверяй памяти. Установи
   факт наличия/отсутствия и точные сигнатуры для:
   - `Hnsw::file_dump(...)` и загрузка `HnswIo` / `load_hnsw_with_dist` (или как
     оно реально называется) — для снапшотов (P2);
   - `parallel_insert(&[(&Vec<T>, usize)])` (или аналог) — для batch (P0/P4);
   - `search_filter(...)` с каким-либо `FilterT`/предикатом — для co-filter (P3);
   - параметризуемость `Hnsw<T, D>` типом `T = i8`/`u8` (компилируется ли граф
     на квантованных векторах) — для P5.
3. **Контракт-тесты** — новый файл
   `crates/shamir-index/src/vector/tests/hnsw_rs_contract_tests.rs`
   (подключить в `crates/shamir-index/src/vector/tests/mod.rs` как `pub mod
   hnsw_rs_contract_tests;` — следуй существующей раскладке tests/; НЕ
   встраивай inline `#[cfg(test)] mod`). Тесты фиксируют РЕАЛЬНОЕ поведение:
   - **dump/load round-trip**: построить маленький `Hnsw` с нашей метрикой
     (`ShamirDist` уже есть в `crates/shamir-index/src/vector/hnsw_adapter.rs` —
     переиспользуй), вставить N векторов, `file_dump` во временную директорию
     (`tempfile`-крейт — сверь, есть ли он в dev-deps; если нет — используй
     `std::env::temp_dir()` + уникальное имя), загрузить обратно с той же
     кастомной метрикой, проверить, что top-k соседи для нескольких запросов
     идентичны до и после;
   - **lifetime `Box::leak`**: зафиксировать паттерн, которым загруженный
     `Hnsw<'static,...>` получается из borrow'ящего загрузчика (в снапшот-коде
     P2 нам нужен `Arc<Hnsw<'static,...>>`); тест компилирует и использует этот
     паттерн (leak маленькой структуры на load — осознанно, boot-only);
   - **parallel_insert эквивалентность**: batch-вставка даёт те же видимые id и
     recall не хуже последовательной;
   - **search_filter / FilterT**: поиск с множеством разрешённых internal id —
     семантика фильтра, поведение при селективности <1% и при пустом множестве;
   - **`Hnsw<i8,_>` компилируемость**: тест-заглушка, доказывающая, что граф
     на `i8`-векторах с подходящей `Distance<i8>` компилируется (для P5).
   - Если КАКОГО-ТО API в 0.3.4 НЕТ — НЕ выдумывай. Задокументируй факт
     комментарием в тесте (`// hnsw_rs 0.3.4: <api> ОТСУТСТВУЕТ — фаза PX
     должна использовать <альтернативу>`) и, где возможно, тестом на реально
     доступную альтернативу. Твой вывод об отсутствии — ценнее ложного теста.
