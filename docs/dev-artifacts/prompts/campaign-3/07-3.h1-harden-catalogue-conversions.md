# ③.h1 — Упрочнить FilterValue↔QueryValue catalogue-конверсии

> Бриф для делегированного агента (`sh`, smart). Follow-up из zero-trust ревью
> ③.2c (`03eff061`). Цель — убрать SILENT-порчу из catalogue-конверсий default.
> НЕ горячий путь (DDL/boot). Поведение НЕ меняем — только убираем тихие провалы.

## ⛔ Запреты
- НЕ `git reset/checkout/clean/stash/restore/rm` и любая git-мутация дерева/индекса.
  Только редактируй; коммитит оркестратор. НЕ удаляй отслеживаемые. НЕ sub-agent.
- Тесты — ТОЛЬКО `./scripts/test.sh`. Хирургически. НЕ меняй наблюдаемое поведение
  на корректных входах (тесты ③.2c/d ОБЯЗАНЫ остаться зелёными).

## Проблема (3 места, все с SILENT fallback)
1. `crates/shamir-query-types/src/filter/filter_value.rs` — `From<QueryValue> for
   FilterValue` через `rmp_serde` round-trip + `.unwrap_or(FilterValue::Null)`:
   литерал QueryValue молча превращается в Null при любом сбое (скрытая порча).
2. `crates/shamir-db/src/shamir_db/execute/admin_schema.rs` —
   `insert_constraint_fields` (write: FilterValue→QueryValue, `if let Ok` молча
   роняет) + `dto_one_from_catalogue` (read: молча `.ok()`).
3. `crates/shamir-db/src/shamir_db/shamir_db/schema_management.rs` —
   `parse_one_rule` (read: молча `.ok()` → default отбрасывается).

## Контекст (уже есть прямой образец)
`crates/shamir-engine/src/validator/schema/schema_validator.rs` —
`filter_value_to_query_value(&FilterValue) -> Option<QueryValue>` (ПРЯМОЙ match,
total по литералам + Array; `_ => None` для выражений). Это эталон обратного
направления.

Ключ: каталог хранит `QueryValue`. ЛИТЕРАЛЬНЫЙ default ↔ прямой match (без
msgpack). ВЫРАЖЕНИЕ-default (`$fn`/`$expr`/`$ref` — FilterValue::{FnCall,Expr,
FieldRef,Cond}) у QueryValue НЕТ варианта → каталог хранит его как
`QueryValue::Map {"$fn":...}`; здесь msgpack round-trip ОПРАВДАН (структурная
сериализация). Так что msgpack убираем ТОЛЬКО для литералов; для выражений
оставляем, но БЕЗ тихого провала.

## Задача
1. **Прямой `query_value_to_filter_value(&QueryValue) -> Option<FilterValue>`**
   (зеркало `filter_value_to_query_value`): total по литералам Null/Bool/Int/
   F64→Float/Str→String/Bin→Binary + List→Array (рекурсивно). Экзотика без
   FilterValue-эквивалента (Dec/Big/Set, если такие есть) → `None`. Положи рядом
   с обратным (или в filter_value.rs). Это убирает msgpack для литералов.
2. **`From<QueryValue> for FilterValue`**: сначала прямой
   `query_value_to_filter_value`; если `None` (Map/экзотика) → msgpack-fallback
   (для выражений-в-Map). Если И msgpack провалился — это developer-ошибка на
   build-time билдера: оставь `FilterValue::Null` ТОЛЬКО как последний резерв,
   но добавь `debug_assert!`/`tracing::warn!` с диагностикой (НЕ молча). Документируй.
3. **READ-пути** (`dto_one_from_catalogue`, `parse_one_rule`): прямой литерал →
   FilterValue; Map → msgpack; при genuine-провале на read — `tracing::warn!`
   («catalogue default decode failed for field X — dropped») вместо тихого `.ok()`.
   boot-resilience сохрани (default → None, не паника), но провал ВИДИМ в логе.
4. **WRITE-путь** (`insert_constraint_fields`): литеральный FilterValue → прямой
   `filter_value_to_query_value`; выражение → msgpack→QueryValue::Map. При
   провале write — `tracing::warn!` (НЕ молча `if let Ok` без else).

⚠ Если `filter_value_to_query_value` сейчас `pub(crate)` в shamir-engine и
недоступен из shamir-db/query-types — НЕ тяни кросс-крейтные зависимости ради
этого; продублируй маленький прямой match локально ИЛИ подними хелпер в
query-types (где живёт FilterValue). Выбери минимально-инвазивное; обоснуй коммментом.

## Тесты (`./scripts/test.sh`)
- РЕГРЕССИЯ: все ③.2c/③.2d тесты зелёные (литерал+выражение default, stamping).
- Новый unit: `query_value_to_filter_value` — литералы прямо, List рекурсивно,
  экзотика→None; симметрия round-trip литерала
  (`query_value_to_filter_value(filter_value_to_query_value(lit)) == lit`).
- (если выразимо) выражение-default по-прежнему персистит+читается через каталог
  (это уже косвенно в ③.2c constraints_dto_tests fn_call round-trip).
