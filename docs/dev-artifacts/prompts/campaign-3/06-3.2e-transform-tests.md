## ЗАДАЧА — два недостающих центра

### 1. Replay-идемпотентность (KEYSTONE) — durable reopen
Доказать инвариант: transforms бегут на ADMISSION, НЕ на WAL-replay
(`VALIDATORS.md:126-131`). После reopen значение НЕ пере-штамповано.

Образец — `crates/shamir-db/tests/rename_db_e2e.rs:248-318`
(`reopen_durable(sys_path)` helper, `tempfile::tempdir()`,
`SystemStoreConfig::Fjall`). ⚠ УРОК ②.1d: используй `tempfile::tempdir()`
(НЕ `std::env::temp_dir()`) — иначе data_root шарится между прогонами и
строки/таймстампы накапливаются (ложный RED). data_root должен жить ВНУТРИ
tempdir, чтобы данные пережили reopen, но были изолированы между прогонами.

Тест (новый файл `crates/shamir-db/tests/declarative_schema_stamping_replay_e2e.rs`
или добавь в существующий stamping_e2e, но reopen требует durable Fjall — см.
образец):
- init durable Fjall DB; set schema с `auto_now` (updated_at) + `auto_now_add`
  (created_at);
- insert строку; прочитай `updated_at = T1`, `created_at = C1`;
- DROP db (закрой); `reopen_durable` на ТОМ ЖЕ sys_path;
- прочитай снова → **assert `updated_at == T1` И `created_at == C1`** (НЕ
  пере-штамповано часами reopen-времени). Это и есть доказательство
  replay-safety.
- (желательно) после reopen сделай UPDATE → updated_at меняется на T2 (>T1),
  созданный заново reopen НЕ ломает stamping на новых write.

### 2. Порядок transform ПЕРЕД CHECK
Доказать: CHECK-валидатор видит ТРАНСФОРМИРОВАННОЕ значение.
- Поле с `default`-выражением/литералом + CHECK-правило на ТОМ ЖЕ поле
  (напр. `one_of [...]` или `format`/`min`): insert БЕЗ поля → default
  заполняет → CHECK валидирует заполненное → проходит (значение default
  валидно). Assert: запись записана, поле = default.
- Негатив (если дёшево выразить): default, нарушающий CHECK → write REJECT
  (доказывает, что CHECK видит post-transform значение, а не отсутствие поля).
  Если негатив сложен в текущем API — оставь коммент-обоснование, не форсируй.
- Уровень: e2e (`shamir-db/tests`) предпочтительно — проверяет всю цепочку
  resolve→apply_defaults→apply_transforms→encode→CHECK.

### 3. (опц., если дёшево) ComputedDefault `$ref` к sibling-полю
eval_write_value резолвит `$ref` по literal-полям записи (apply_transforms
снимок map). Тест: default = `$ref('other_field')`, insert с other_field →
computed-default копирует значение. Если builder/API не даёт удобно выразить
$ref-default — пропусти с комментом.
