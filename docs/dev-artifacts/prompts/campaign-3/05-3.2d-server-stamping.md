# ③.2d — Server-stamping `created_at` / `updated_at`

> Бриф для делегированного агента (`sh`, smart). Едет ПОВЕРХ ③.2b (`e77ec918`)
> и ③.2c (`03eff061`). Источник: `CAMPAIGN-3-PLAN.md §«③.2a — boundary»`,
> `TransformSpec::{AutoNowAdd, AutoNow}` (уже определены в ③.2b). Здесь —
> SCHEMA-SURFACE (флаги) + UPDATE-path wiring. Решения развилок ПРИБИТЫ.

## ⛔ Запреты
- НЕ `git reset/checkout/clean/stash/restore/rm` и любая git-мутация дерева/индекса.
  Только редактируй; коммитит оркестратор. НЕ удаляй отслеживаемые. НЕ sub-agent.

## Прибитые решения (НЕ пере-решай)
1. **Семантика по op:** `AutoNow` (`updated_at`) штампуется БЕЗУСЛОВНО на КАЖДОМ
   write (insert И update). `AutoNowAdd` (`created_at`) и `ComputedDefault` —
   ТОЛЬКО на insert (absence-guarded на ПОЛНОЙ записи). На update частичный
   `set`-map не содержит created_at → AutoNowAdd там WRONGLY заштамповал бы →
   ОБЯЗАН пропускаться.
2. **Механизм гейта:** `apply_transforms` получает НОВЫЙ параметр
   `is_insert: bool`. AutoNow — всегда; AutoNowAdd + ComputedDefault — только при
   `is_insert`. Обнови insert-вызов (#280, write_exec.rs:~170) на `is_insert=true`
   и тесты ③.2b (`apply_transforms_tests.rs`) под новую сигнатуру.
3. **Тип штампа:** `QueryValue::Int(now_ns as i64)` (как ③.2b). Не вводи новый тип.

## Задача
1. **Surface-флаги** в `ConstraintsDto` (`shamir-query-types/src/admin/types/schema_ops.rs`)
   + `Constraints` (`shamir-engine/src/validator/schema/constraints.rs`):
   `auto_now: bool` и `auto_now_add: bool` (serde default false; зеркаль стиль
   существующего bool-флага `unique`). Прокинь через catalogue read/write
   (`admin_schema.rs` insert_constraint_fields/dto_one_from_catalogue,
   `schema_management.rs` parse_one_rule) — как `unique`.
2. **Билдеры:** Rust `FieldBuilder` (`shamir-query-builder/src/ddl/schema.rs` +
   engine `rule_builder.rs`) и TS (`shamir-client-ts/src/core/builders/ddl.ts` +
   тип `core/types/ddl.ts`): методы `.auto_now()`/`.auto_now_add()` (Rust,
   snake как соседи) и `.autoNow()`/`.autoNowAdd()` (TS camelCase). JSDoc/doc:
   created_at=auto_now_add (insert-only), updated_at=auto_now (каждый write).
3. **transforms()** (`schema_validator.rs`, ③.2c уже реализовал для ComputedDefault):
   дополнительно эмить `(path, AutoNowAdd)` для правил с `auto_now_add=true` и
   `(path, AutoNow)` для `auto_now=true`.
4. **apply_transforms `is_insert`-гейт** (`write_helpers.rs`): см. реш. 2.
5. **UPDATE-path wiring** (`write_exec.rs` `execute_update_tx`): ПОСЛЕ
   `resolve_computed_record(&op.set)` (~:355), ДО конверсии в inner, вызови
   `apply_transforms(resolved_set.to_mut(), &transforms, builtin_scalars(),
   now_ns, /*is_insert=*/false)` с fast-skip `if !transforms.is_empty()`.
   `transforms = self.schema_transforms()`, `now_ns` из SystemTime (как insert).
   На update сработает ТОЛЬКО AutoNow (updated_at инжектится в set).
   ⚠ Сверь, что `resolved_set` — `QueryValue::Map` и `.to_mut()` доступен.
6. **UPSERT-path** (`write_exec.rs:~774`, `resolve_computed_record(&op.value)`):
   value — ПОЛНАЯ запись (insert-семантика) → вызови с `is_insert=true`. Если
   форма value неоднозначна — НЕ угадывай, оставь TODO-коммент и сообщи.

## Тесты (`./scripts/test.sh @engine`, НЕ raw cargo)
- insert: и created_at, и updated_at заштампованы.
- update: ТОЛЬКО updated_at заштампован (created_at НЕ тронут — проверь, что
  существующий created_at в записи не изменился после update).
- auto_now перезаписывает caller-значение updated_at; auto_now_add сохраняет
  явный created_at.
- apply_transforms_tests: добавь is_insert=false кейс (AutoNowAdd/ComputedDefault
  пропущены, AutoNow сработал).
- TS: vitest ddl.test + tsc для новых .autoNow()/.autoNowAdd().
