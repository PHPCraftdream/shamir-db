# Zero-capacity campaign — pre-size every growable collection

Цель (ideology pillar 3, `O(x→0)`): ни одна **растущая** коллекция не создаётся
с нулевой ёмкостью. `Vec`/`VecDeque`/map/set, начатые пустыми, при росте
реаллоцируются **удвоением** (`alloc` + `memcpy` снова и снова) — «мучение кучи».
Каждая такая коллекция создаётся с **оценённой** ёмкостью.

Объединяет это с pillar 4 (Fx-only hashing): для мап/сетов единственная
правильная форма — `with_capacity_and_hasher(n, THasher::default())` (Fx + размер).

---

## 1. Единственно разрешённые формы создания

| Структура | Разрешено | Запрещено |
|---|---|---|
| `Vec` / `VecDeque` | `::with_capacity(n)` | `::new()`, `::default()`, пустой `vec![]`, `with_capacity(0)` |
| `TMap` / `TSet` (IndexMap/IndexSet + Fx) | `shamir_collections::new_map_wc(n)` / `new_set_wc(n)` | `new_map()`, `new_set()`, `::with_hasher()`, `::default()` |
| std `HashMap` / `HashSet` | `::with_capacity_and_hasher(n, THasher::default())` | `::new()`, `::with_capacity()`, `::with_hasher()`, `::default()` |
| `DashMap`, `scc::HashMap` | `::with_capacity_and_hasher(n, THasher::default())` | `::new()`, `::with_hasher()`, `::with_capacity()`, `::default()` |

**Хелперы уже есть** в `shamir-collections/src/lib.rs`:
`new_map_wc(n)` = `IndexMap::with_capacity_and_hasher(n, THasher::default())`,
`new_set_wc(n)` (+ `new_fx_map_wc` / `new_fx_set_wc` для std-HashMap-on-Fx).
Их zero-cap аналоги (`new_map()` / `new_set()` / `new_fx_*`) — под запрет у вызывающих;
сами определения хелперов несут **один санкционированный** `#[allow(clippy::disallowed_methods)]`.

### Как оценивать `n`
- push/insert в цикле по `X` → `X.len()`.
- известное число элементов → это число.
- ограничено входом → длина входа (верхняя граница).
- размер непредсказуем, но коллекция растёт → realistic типовой объём (4 / 8 / 16); лёгкий перебор лучше нуля.

### String — ИСКЛЮЧЁН
`String::new()` НЕ банится. Большинство строк потом переприсваиваются/перезаписываются
(`let mut s = String::new(); s = format!(...)`) — пресет ёмкости там бессмысленен.
Где строка реально растёт через `push_str`/`write!` — `String::with_capacity(bytes)`
приветствуется, но это не enforced.

### Переприсваивание (важный нюанс)
Если переменная создаётся пустой и сразу **перезаписывается** значением, ёмкость
бессмысленна → правильно **реструктурировать**: инициализировать сразу значением,
а не «пусто, потом присвоить». Не лепить `with_capacity` на выброшенный буфер.

## 2. NO DODGES (запреты обхода)

Единственный приемлемый фикс — реальный `with_capacity*` / `new_map_wc` / `new_set_wc`.
Запрещены ВСЕ обходы:
- `#[allow(clippy::disallowed_methods)]` (кроме санкционированных хелперов в shamir-collections);
- `::default()` для Vec/VecDeque/map/set;
- пустой `vec![]`;
- `with_capacity(0)` / `with_capacity_and_hasher(0, …)`;
- `with_hasher(…)`, `new_map()`, `new_set()`.

Агенты на прошлых попытках **гейлили** именно так (массовый `#[allow]`, замена на
`::default()` и пустой `vec![]`). Это отклоняется на ревью диффа (`grep` по добавленным
доджам).

---

## 3. Что clippy РЕАЛЬНО ловит, а что — ревью

`clippy.toml` `disallowed-methods` может банить только **inherent** методы по пути:
`::new`, `::with_hasher`, `::with_capacity`, проектные хелперы (`new_map`/`new_set`).

**clippy НЕ может** (ловится ревью диффа грепом):
- `::default()` — это трейт-метод `Default::default`, по типу не банится (иначе бан всех `.default()`);
- пустой `vec![]` (раскрывается в `Vec::new`) — флагается, но «fix» = реструктур/документировать;
- `with_capacity(0)`;
- `#[allow(clippy::disallowed_methods)]`.

## 4. Неустранимые стены (почему чистый `-D` нереален без оговорок)

Замер показал ~2065 уникальных флагов (`Vec::new` 1070, `new_map` 1142, `scc/Dash::with_hasher` ~50).
**1106 из них — макро-раскрытия** (892 `mpack!`, 200 `vec!`). Разбор стен:

1. **serde `#[derive(Deserialize)]`** генерит `Vec::new()` в СВОём сгенерированном
   коде для каждого `Vec`-поля → clippy флагает строку `#[derive(...)]`
   (`query_entry.rs`, `tables_registry.rs`, и так — каждый wire-DTO с Vec-полем).
   Это **чужой код**, через `with_capacity` не чинится. Только `#[allow]` или warn.
2. **Genuinely-empty возвраты** (`Ok(Vec::new())`, `return vec![]`, `tokens: vec![]`)
   — у пустой коллекции **нет** осмысленной ёмкости (тот же инсайт, что про строки).
3. **Наши макросы** (`mpack!`, `doc!`, `vals!`) раскрываются в `new_map()`/`vec!` —
   чинить нужно **определение макроса** (пресайз по числу элементов), а НЕ сотни вызовов.
   clippy нельзя настроить смотреть код «до макросов» — `disallowed_methods` срабатывает
   на развёрнутом коде и не пропускает ни внешние, ни наши макросы.

Вывод: `-D`-зелёный достижим только с **точечными `#[allow]`** на serde-derive'ах и
genuinely-empty (физически единственный путь), либо лит на уровне **warn**.

## 5. План исполнения (с пошаговыми коммитами — НЕ терять работу)

> Урок: на больших объёмах **коммить промежуточное состояние ПЕРЕД любым откатом**.
> Один незакоммиченный `git checkout HEAD -- crates/` уже стёр ~850 готовых `with_capacity`.

> Урок (модель): в `Workflow` `agent()` берёт модель из **главного цикла сессии**
> (сейчас Opus 4.8), а `agentType:'ash'` меняет ТОЛЬКО системный промпт. Чтобы реально
> Sonnet — указывать `model:'sonnet'` в каждом `agent()`. Иначе sweep идёт на дорогом Opus.

1. **clippy.toml** — вернуть zero-cap бан (Vec::new, VecDeque::new + мап-формы
   `new_map`/`new_set`/`with_hasher` для std/Dash/scc; String исключён) → **коммит**.
2. **Наши макросы** (`mpack!`/`doc!`/`vals!`) — пресайз по числу элементов
   (подсчёт в `macro_rules!` через `<[()]>::len(&[$(unit!($k)),*])` или `${count(...)}`);
   пустые арки оставить. → **коммит**.
3. **@ash sweep (`model:'sonnet'`)** по реальным сайтам (точные адреса из свежего
   `clippy` → todo-файл), партиции по крейтам/подкаталогам, строгий анти-додж промпт,
   `new_map()→new_map_wc(n)`, `Vec::new()→with_capacity(n)`. → **коммит по батчам**.
4. **Неустранимое** (serde-derive + genuinely-empty) — точечный `#[allow]` с
   `// reason:` (задокументировать в этом файле список категорий) ИЛИ лит на warn.
   → итог `clippy -D` зелёный + полный гейт (check/clippy/fmt/test/@e2e). → **коммит**.

### Альтернатива (если blanket окажется не стоящим)
Реальная ценность — **горячие grow-коллекции** (циклы, растущие большими на
per-row/per-request путях), их ~десятки-сотня, не 2065. Прицельный проход по ним +
пресайз макросов даёт основной perf-выигрыш без serde/empty-churn. Остальное —
лит на warn как guidance.

## 6. Как брифовать агентов (anti-dodge промпт — переиспользуемый)

Уроки трёх провальных попыток: агенты (включая Sonnet high) **гейлят линт**, а не
делают работу — массовый `#[allow]`, замена `::new()`→`::default()`, пустой `vec![]`,
ручное переписывание сотен вызовов макроса вместо фикса определения. Поэтому промпт
должен быть бритвенно-точным, а **ревью диффа на доджи — обязательным**.

### Что ОБЯЗАТЕЛЬНО в промпте агента
- **Точные адреса, не «грепни сам».** Дать `.zerocap-todo.txt` (`crates/<crate>/<path>:line:col`
  из свежего `cargo clippy`); «почини КАЖДУЮ строку под своим крейтом».
- **Единственно приемлемый фикс** — реальный `with_capacity(n)` / `new_map_wc(n)` /
  `new_set_wc(n)` / `with_capacity_and_hasher(n, THasher::default())`.
- **Явный список запрещённых доджей** с пометкой «работа будет ОТКЛОНЕНА на ревью»:
  `#[allow(clippy::disallowed_methods)]`, `::default()`, пустой `vec![]`,
  `with_capacity(0)`, `with_hasher()`, `new_map()`/`new_set()`.
- **Вдумчивость по каждому сайту** (это суть, НЕ механика — иначе шум):
  - растёт в цикле → `with_capacity(<bound>)` (`src.len()` / известное число / `input.len()`).
  - создаётся-и-сразу-переприсваивается → **реструктур** к прямой инициализации (ёмкость там бессмысленна).
  - реально пустой / sentinel / возврат-пустого → реструктур, либо (serde/empty-кромки) задокументированный `#[allow]`.
- **String исключён.** `vec![a, b]` С элементами — ОК (уже преразмерен).
- **Наши макросы чинить в ОПРЕДЕЛЕНИИ**, а не в вызовах (см. §4.3). Агенту по vec/map-сайтам
  запретить трогать `mpack!`/`doc!`/`vals!`-вызовы — их закрывает фикс макроса.
- **Без cargo.** Агент только правит; гейт у оркестратора. Не читать `.claude/worktrees/`.

### Модель (критично по стоимости)
В `Workflow` `agent()` указывать **`model:'sonnet'`** в КАЖДОМ вызове — иначе агент
наследует модель главного цикла сессии (сейчас Opus 4.8 — дорого). `agentType:'ash'`
меняет ТОЛЬКО системный промпт субагента, **не модель**.

### Партиции
Гиганты (engine ~780, db ~230, builder ~270) бить по подкаталогам (`table/`, `tx/`,
`query/`, …), ≤~70 сайтов на агента — иначе качество падает и агент недобивает.

### Оркестратор ОБЯЗАН (после каждой волны)
1. **Анти-додж-греп диффа**: `git diff | grep '^+'` на `::default()` / `vec!\[\]` /
   `with_capacity(0)` / `with_hasher` / `#[allow(...disallowed_methods)]`
   (кроме санкционированных хелперов в shamir-collections). Любой додж → откат+передел этого батча.
2. **Коммит промежуточного состояния ПЕРЕД любым `git checkout`/откатом** (иначе потеря, как с ~850 правками).
3. Гейт: `cargo check` → `clippy -D warnings` → `fmt --check` → `./scripts/test.sh` → `@e2e`.
4. Финальный **независимый** ревью (Opus, 1 агент) — осмысленность ёмкостей, скрытые доджи.
