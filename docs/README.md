# ShamirDB documentation

This directory contains the project's user guides, architecture notes, protocol specifications, design records, operational notes, audits, and roadmap material.

## Start here

- [Quick Start](guide/00-quickstart.md) — run the server and make the first request.
- [Guided documentation](guide/README.md) — progressive tour from basic queries to operations and interconnection.
- [Architecture](architecture/ARCHITECTURE.md) — storage, execution, transactions, indexes, and security boundaries.
- [Logic flow](architecture/LOGIC_FLOW.md) — how a request moves through the system.
- [Protocol specification](client-server-protocol-spec/README.md) — authentication, sessions, subscriptions, and transports.
- [Security and data protection](security/data-protection.md) — current guarantees and limitations.
- [Roadmap](roadmap/ROADMAP.md) — planned work and known gaps.

## By topic

- `design/` — design decisions and implementation plans.
- `ops/` — operating, capacity, and performance guidance.
- `audits/` — security, performance, and architecture reviews.
- `benchmarks/` and `perf/` — benchmark methodology and measured results.
- `checkpoints/` — historical development notes; these are not release notes.

Some documents describe proposed or incomplete work. Their status is authoritative only when the document explicitly says that the feature is implemented and the code and tests agree.
