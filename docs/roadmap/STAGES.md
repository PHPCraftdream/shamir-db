# Этапы после MVP — карта развития

**Status:** living roadmap, revision **2026-06-02**.

MVP готов и проверен: durable KV, SCRAM-аутентификация, иерархический доступ
(Shomer), хранимые WASM-функции, durability-уровни (`buffered`/`synced`),
graceful shutdown, простой старт (см. `docs/guide/00-quickstart.md`). Это
самодостаточная СУБД. Ниже — путь от MVP к production-продукту.

## Этап A — Эксплуатация (как это деплоить) ← ТЕКУЩИЙ

Чтобы MVP реально выкатывался в прод.

* **#115 Runtime modes** — один бинарник, но и foreground («как запустят»), и
  сервис (systemd на Linux / Windows-service), с корректной остановкой.
  Фундамент готов: **#114 graceful shutdown** закрыт (дренаж репо-буферов +
  scheduler + audit).
* **#116 Logging** — неблокирующее, батчевое, с неймспейсами/масками
  (`docs/roadmap/LOGGING.md`).

## Этап B — Доступ для бизнеса (завершить multi-tenant)

* `ctx.caller` / `ctx.can` / row-level — процедуры исполняются под логином
  вызвавшего юзера; быстрая проверка прав на ресурс из процедуры. (Отложено в
  созерцании — см. историю.)
* **#118 Папки функций** — `ResourcePath::FunctionFolder`: права на наборы
  процедур как на POSIX-директории.
* Итог: «бизнес-пользователи работают только через свои setuid-процедуры».

## Этап C — Документация (adoption)

* **#124** — спиральный гайд, этажи 1–8 (`docs/guide/`). Этаж 0 готов и
  проверен e2e; этажи наполняются поэтажно с заземлением примеров на тесты.

## Этап D — Поиск / AI (продуктовые фичи)

* **#110** — FTS: стемминг по языкам Snowball + Ngram.
* Векторы (HNSW уже частично есть) — embeddings/семантический поиск.

## Этап E — Производительность (по измеренному сигналу)

* **#122** group-commit (амортизировать fsync `synced` под нагрузкой) и
  опти-циклы — делать, когда бенч покажет узкое место, не спекулятивно.

## Этап F — Interconnect (the «I»)

* P2P / chat — децентрализация. Большая будущая веха, отдельный design-doc.

## Рекомендованный порядок

A (эксплуатация) → параллельно C (доки) → B (бизнес-доступ) → D (поиск) →
E (перф, по сигналу) → F (P2P). A первым: без сервис-режима и логов MVP не
задеплоить, а фундамент (#114) уже стоит.
