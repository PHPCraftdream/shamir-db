# S.H.A.M.I.R. — Спиральный путеводитель

Документация по **этажам**. На первом — самое простое: за пять минут
получаешь работающее KV-хранилище. Каждый следующий этаж добавляет
**ровно столько** деталей, сколько нужно растущему продукту — по мере того
как он постепенно, без переписывания, **полностью переезжает на Shamir DB**.

Не нужно читать всё. Поднимайся на этаж только когда упёрся в его потребность.

```
этаж 8  Interconnect (P2P / chat)            ← когда нужна децентрализация
этаж 7  Эксплуатация (service, метрики, логи) ← когда выкатываешь в прод
этаж 6  Поиск (FTS, векторы / embeddings)     ← когда нужен поиск/AI
этаж 5  Функции (WASM, the "M")               ← когда логика переезжает в БД
этаж 4  Доступ (Shomer: юзеры, группы, права) ← когда пользователей много
этаж 3  Хранилища (репозитории, бэкап)        ← когда данных много
этаж 2  Durability и транзакции               ← когда терять нельзя
этаж 1  Запросы (фильтры, индексы, батчи)     ← когда KV становится мало
этаж 0  Quickstart — KV за 5 минут            ← старт здесь
```

## Карта этажей

| Этаж | Файл | Что освоишь | Когда подниматься |
|---|---|---|---|
| **0** | [`00-quickstart.md`](./00-quickstart.md) | Один бинарник, аккаунт, таблица в `default`, put/get. Работает как KV. | Сразу. |
| **1** | `01-queries.md` | Фильтры (`where`), батчи с зависимостями, вторичные индексы, диапазоны/сортировка/`LIMIT`. | KV мало — нужны выборки. |
| **2** | `02-durability.md` | `durability: buffered`/`synced`, транзакционные батчи + `isolation` (snapshot/serializable), graceful shutdown. | Данные терять нельзя. |
| **3** | `03-storage.md` | Репозитории (durable-by-default), `in_memory`-scratch, бэкап, интроспекция `access-tree`. | Несколько хранилищ / много данных. |
| **4** | `04-access.md` | Shomer: юзеры, группы, POSIX-режимы (`chmod`/`chown`/`chgrp`), права на ресурсы, бизнес-доступ через процедуры. | >1 типа пользователей. |
| **5** | `05-functions.md` | Хранимые WASM-функции (`#[shamir::function]`), `ctx` (db/call/http), setuid-процедуры. | Логика переезжает в БД. |
| **6** | `06-search.md` | Full-text search (токенизация, стемминг), векторы/embeddings (HNSW). | Нужен поиск / AI-фичи. |
| **7** | `07-operations.md` | Рантайм-режимы (foreground + service/systemd, один бинарник), `/healthz` `/readyz` `/metrics`, логирование, rate-limit, конфиг. | Выкатка в прод. |
| **8** | `08-interconnect.md` | P2P / chat (the "I"). | Децентрализация. |

## Сквозной принцип

Истина живёт в **одном** месте — версионированном MVCC-сторе. Всё
производное (кеши, индексы, проекции) — overlay над ней, восстановимый из
WAL. На каждом этаже это всплывает по-новому, но модель одна. Подробности —
по мере подъёма.

> Этажи 1–8 наполняются по ходу (см. задачу в роадмапе). Этаж 0 — готов и
> проверен на реальном пути из `crates/shamir-server/tests/mvp_e2e.rs`.
