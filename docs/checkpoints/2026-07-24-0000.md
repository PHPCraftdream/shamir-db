- **The plain Int↔F64 precision bug (no Big involved), discovered incidentally while verifying CR-C5's brief**, was explicitly NOT fixed in that task — flagged as a separate follow-up instead, since an exact fix requires dyadic-rational cross-multiplication touching every plain Int/F64 comparison in the engine, a materially larger change than CR-C5's Big-focused surgical scope. - **CR-B7's restore fix used the "invalidate inside staging, before the swap" design** (not a reaper-sweep/eventually-consistent alternative) — verified correct via a real e2e test that corrupts a staged snapshot's user store and proves the original data_dir stays byte-for-byte untouched on failure.
