# Prompts — исходник исходника

Здесь живут **брифы** (промпты) делегированных этапов работы: то, что
передаётся агенту (`/crush`, `o46l` и пр.) как задача. Брифы версионируются
в репозитории намеренно.

## Зачем

Урок, оплаченный кровью (2026-06-24): агент выполнил ad-hoc `git reset --hard`
и стёр часы незакоммиченной работы. Восстановление оказалось **возможным** ровно
потому, что уцелели две вещи: **брифы** и **тест-файлы**. Код — эфемерен; его
можно переродить. Долговечны:

> **промпт (что хотели) + тест (как проверяем) = воспроизводимая фича.**

Поэтому промпт — первоклассный артефакт репозитория, не временный файл в
`.crush/` (который в `.gitignore`).

## Дисциплина prompt-first (нормативно — см. CLAUDE.md)

1. **Перед раном этапа** (любой делегированный агент / фаза) — записать его бриф
   сюда, в `docs/prompts/<область>/<NN>-<имя>.md`, и **закоммитить**.
2. Только потом запускать агента.
3. Сгенерированный код + тесты — отдельным коммитом (или серией) после
   верификации оркестратором.

Так каждая фича переродима из своего брифа, ревью «замысел» отделено от
«реализации», а авария уровня reset превращается из катастрофы в `re-run`.

## Запрет для агентов

В бриф КАЖДОГО делегированного рана включать жёсткий запрет:
**NEVER `git reset` / `checkout` / `clean` / `stash` / `restore` / `rm`** —
агент работает только правками файлов; коммитит оркестратор. (Корень аварии —
агент тронул git-состояние.)
