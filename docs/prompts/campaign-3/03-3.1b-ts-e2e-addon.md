# ③.1b — TS e2e-добивка + tsc-долг (server-gated)

> Бриф для делегированного агента (`sh`). Источник: `CAMPAIGN-3-PLAN.md §③.1b`,
> `coverage-ts-tests.md §6`. ⚠ Аудит-источник УСТАРЕЛ: многие e2e уже есть
> (`e2e-fts/vector/call/resume/data/ddl/...`). Не дублируй — заполняй РЕАЛЬНЫЕ
> пробелы. Главная верифицируемая (server-free) часть — почини tsc-долг.

## ⛔ Запреты
- НЕ `git reset/checkout/clean/stash/restore/rm` и любая git-мутация дерева/индекса.
  Только редактируй; коммитит оркестратор. НЕ удаляй отслеживаемые. НЕ sub-agent.
- Скоуп: `crates/shamir-client-ts/`. Хирургически.
- ⚠ НЕ запускай rust `--full`/сервер параллельно (Windows file-lock на
  `shamir-server.exe`) — здесь это и не нужно (vitest без сервера skip'ает e2e).
- Запросы строй ТОЛЬКО через билдер (write/query/filter/ddl), НЕ сырой объект.

## Часть A (server-free, ВЕРИФИЦИРУЕМО) — почини tsc-долг
`npx tsc --noEmit` даёт 4 ошибки в `src/__tests__/e2e-schema-validators.test.ts`
(строки ~55/82/109/110): `TS2345 Argument of type 'Record<string, unknown>' is
not assignable to 'WriteValue | WriteValue[]'`. Причина — сырые объекты записей
вместо типобезопасного построения. Почини, используя write-билдер (как в соседних
e2e — посмотри, как `e2e-data.test.ts`/`e2e-ddl.test.ts` строят insert-записи
через билдер). НЕ ослабляй типы кастами `as`; построй значения корректно. После —
ровно НОЛЬ tsc-ошибок.

## Часть B (server-gated) — РЕАЛЬНЫЕ пробелы, не дубли
1. **Сначала сверься с фактами:** перечитай существующие `src/__tests__/e2e-*.test.ts`
   и сопоставь с `coverage-ts-tests.md §6` (P0 FTS/vector/call; P1 like/ilike/regex,
   isNull/exists/contains*, aggregateFn, func scalar-projection, history-range,
   page-mode, distinct; P2/P3 resume/commitMigration/dropUser/dropRole).
   Многое УЖЕ покрыто (`e2e-fts`, `e2e-vector`, `e2e-call`, `resume.test.ts`, …).
2. **Добавляй ТОЛЬКО genuine-пробелы** под `describe.skipIf(!SERVER_AVAILABLE)`
   (паттерн — `connect.test.ts:33`, `e2e-call.test.ts:35`; харнесс —
   `src/__tests__/e2e-harness.ts`). Клади кейсы в подходящий существующий
   `e2e-*.test.ts` (напр. фильтр-операторы like/ilike/regex/isNull/contains* — в
   `e2e-data.test.ts`); новый файл — только если темы нет вовсе.
3. В финальном отчёте ЯВНО раздели: «уже покрыто (файл:describe)» vs «добавлено
   мной». Если P0/P1-пункт уже покрыт — так и напиши, НЕ создавай дубль.

## Гейт (прогони сам, приложи вывод)
```
cd crates/shamir-client-ts
npx tsc --noEmit                                  # ОЖИДАЕМО: 0 ошибок (Часть A)
npx vitest run                                    # зелёно; e2e skip без сервера
```
(Release-сервера у тебя, вероятно, нет — e2e-кейсы должны КОМПИЛИРОВАТЬСЯ и
skip'аться, не падать. Это нормально и достаточно для server-gated этапа.)
Финальным сообщением: изменённые файлы; что починил в Части A; таблица
«покрыто vs добавлено» по Части B; вывод tsc + vitest.
