## Задача (insert-path фреймворк)

### 1. Тип `TransformSpec` (закрытый перечень) — новый файл
`crates/shamir-engine/src/validator/transform_spec.rs` (one-export rule),
re-export через `validator/mod.rs`:
```rust
/// Декларативная transform-операция над записью, применяемая ДО encode
/// (admission-time, НЕ на WAL-replay — replay-safety бесплатна).
pub enum TransformSpec {
    /// ③.2c: вычисляемый default. Применять ТОЛЬКО к отсутствующему полю,
    /// вычислять выражение через `eval_write_value`.
    ComputedDefault(shamir_query_types::filter::FilterValue),
    /// ③.2d created_at: штамп текущего времени на INSERT, только если поле
    /// отсутствует (insert-only).
    AutoNowAdd,
    /// ③.2d updated_at: штамп текущего времени БЕЗУСЛОВНО (перезапись).
    AutoNow,
}
```
(Точный путь к `FilterValue` сверь по импортам `write_helpers.rs` —
`eval_write_value` уже его принимает.)

### 2. `RecordValidator::transforms()` — trait-метод, близнец `defaults()`
В `record_validator.rs` рядом с `defaults()` (:186):
```rust
/// Декларативные transform-правила этого валидатора. Default — пусто
/// (native/WASM не несут transforms; только SchemaValidator переопределит
/// в #281/#282). Зеркало `defaults()`.
fn transforms(&self) -> Vec<(Vec<String>, super::TransformSpec)> { Vec::new() }
```
В `SchemaValidator` (schema_validator.rs) добавь impl `transforms()`, возвращающий
ПОКА `Vec::new()` (surface для сбора появится в #281/#282 — оставь TODO-коммент,
ссылающийся на них). НЕ добавляй поля в schema-rule здесь.

### 3. `TableManager::schema_transforms()` — аггрегатор, близнец `schema_defaults()`
В `table_manager_validators.rs` рядом с `schema_defaults()` (:376): идентичная
структура, но `validator.transforms()`. Возврат `Vec<(Vec<String>, TransformSpec)>`.

### 4. `apply_transforms` — в `write_helpers.rs` рядом с `apply_defaults`
```rust
pub(super) fn apply_transforms(
    rec: &mut QueryValue,
    transforms: &[(Vec<String>, TransformSpec)],
    scalars: &ScalarRegistry,
    now_ns: u64,
) { … }
```
Семантика (insert-path):
- Только `QueryValue::Map` (иначе return, как apply_defaults).
- Single-segment path (как apply_defaults MVP); multi-segment — skip.
- `AutoNow` → `m.insert(field, QueryValue::Int(now_ns as i64))` БЕЗУСЛОВНО.
- `AutoNowAdd` → стамп `QueryValue::Int(now_ns as i64)` только если
  `!m.contains_key(field)`.
- `ComputedDefault(expr)` → только если `!m.contains_key(field)`: вычислить
  `eval_write_value(expr, <literal=текущий map>, scalars)`; при `Ok(v)`
  `m.insert(field, v)`; при `Err` — пропусти молча (fail-open, как scalar-bridge
  прецедент) ИЛИ верни ошибку — выбери fail-open для консистентности с
  ②.4c-стилем (НЕ паникуй). Документируй выбор коммментом.
  (`literal` для eval_write_value — это сам record-map как `&TMap<String,QueryValue>`;
  сверь тип: если `m` уже `TMap`, передай `&*m`/клон по месту вызова сигнатуры.)

### 5. Wiring на insert-пути (`write_exec.rs`)
Рядом с `let defaults = self.schema_defaults();` (:99) добавь
`let transforms = self.schema_transforms();`. После `apply_defaults` (:152), ВНУТРИ
того же per-record цикла, ДО encode (:154):
```rust
if !transforms.is_empty() {
    apply_transforms(resolved.to_mut(), &transforms, <scalars>, now_ns);
}
```
- `now_ns`: вычисли ОДИН раз на батч из `std::time::SystemTime::now()
  .duration_since(UNIX_EPOCH)` → nanos as u64 (off-replay; admission-time).
  Прокинь в цикл. (Узкий clock-источник без инъекции — apply_transforms чист и
  тестируется напрямую с фикс. now_ns.)
- `<scalars>`: возьми тот же ScalarRegistry, что доступен на этом пути
  (сверь, как `resolve_computed_record`/`eval_write_value` его получают — вероятно
  через self или ctx). Если на insert-пути реестр недоступен без усилий —
  передай его так же, как resolve_computed_record получает interner; не изобретай.
- fast-skip `if !transforms.is_empty()` — горячий путь (нет transforms) не платит.
- Поскольку `schema_transforms()` пока возвращает пусто (нет surface), wiring
  ИНЕРТЕН до #281/#282 — это ожидаемо и корректно.

### 6. Unit-тесты — прямой тест `apply_transforms`
Заведи тесты по правилам организации (`tests/`-директория модуля,
`#[cfg(test)] mod tests;`, файл по топику — напр.
`table/tests/apply_transforms_tests.rs` или рядом с существующими write_helpers-тестами;
сверь, где лежат тесты apply_defaults, и положи рядом). Покрой ПРЯМЫМ вызовом
`apply_transforms` с hand-built specs и фикс. `now_ns`:
- `AutoNow` перезаписывает существующее значение на now_ns.
- `AutoNow` штампует отсутствующее.
- `AutoNowAdd` штампует только отсутствующее; present — не трогает.
- `ComputedDefault(литерал)` штампует отсутствующее; present — не трогает.
- `ComputedDefault($fn)` вычисляется через scalars (если есть удобный scalar в
  тест-фикстуре; иначе литеральный кейс достаточен + коммент).
- non-Map запись — no-op.
- пустой `transforms` — no-op (хотя wiring сам fast-skip'ает).
