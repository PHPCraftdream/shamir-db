# Rules for ShamirDB Development

## Core Principles

1. **Do not implement anything without explicit user request**
   - Always ask before adding new code, files, or features
   - Do not make assumptions about architecture

2. **Confirm architectural decisions**
   - Discuss structural changes before implementing
   - Get approval on data structures and interfaces

3. **Clarify requirements**
   - If something is unclear, ask the user
   - Do not guess or make assumptions

## Workflow

1. When asked to implement something:
   - Ask clarifying questions if needed
   - Outline the approach briefly
   - Wait for approval before coding

2. If you encounter uncertainty:
   - Stop and ask the user
   - Do not work around it with placeholders

3. Testing:
   - Only add tests if explicitly requested
   - Follow existing test patterns in the codebase
